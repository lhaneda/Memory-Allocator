Detailed Description

Defines metadata structure for both memory 'regions' and 'blocks.' This structure is prefixed before each allocation's data area.

Field Documentation

◆ alloc_id

unsigned long mem_block::alloc_id

Each allocation is given a unique ID number. If an allocation is split in two, then the resulting new block will be given a new ID.

◆ name

char mem_block::name[32]

The name of this memory block. If the user doesn't specify a name for the block, it should be auto-generated based on the allocation ID.

◆ region_size

size_t mem_block::region_size

If this block is the beginning of a mapped memory region, the region_size member indicates the size of the mapping. In subsequent (split) blocks, this is undefined.

◆ region_start

struct mem_block* mem_block::region_start
